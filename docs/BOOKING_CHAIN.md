# Booking chain and logging

## Как работает оплата (простыми словами)

### Главное: pending **не занимает** места

Когда клиент заполнил форму и создалась бронь **«Ожидает оплату»**:

- Места **ещё не заняты** — в вместимость идут только **подтверждённые** брони.
- Много человек **одновременно** могут создать бронь и открыть Stripe на одну дату.
- Новую бронь блокируем только когда `подтверждённые места >= вместимость`.

### Когда места реально закрываются

1. **Оплата прошла** → бронь `Подтверждена` → места заняты.
2. Если `подтверждённые >= вместимость` (например 40/40):
   - дата помечается закрытой;
   - **все остальные** pending-брони на эту дату отменяются (`sold_out`);
   - их Stripe Checkout сессии принудительно **expire** через API.

Это основная бизнес-логика. Код: `close_competing_sessions_if_sold_out()`.

### Зачем тогда Stripe `expires_at` (24 часа)

Stripe **обязывает** передавать `expires_at` при создании Checkout Session. Минимум 30 мин, максимум 24 ч.

Мы ставим **24 часа** — это технический срок жизни ссылки Stripe, **не** «резерв мест» и **не** настраивается в `.env`.

Когда Stripe сам закрывает сессию через 24 ч → webhook `checkout.session.expired` → эта конкретная pending-бронь помечается истёкшей. На других людей и на вместимость это не влияет.

### Чего больше нет

- `BOOKING_HOLD_MINUTES` в `.env` — удалено
- Таймер 30:00 в форме бронирования — удалён (он ошибочно отменял бронь)
- Celery `expire_pending_bookings` — удалён

### Цепочка по шагам

1. Форма → `create_booking_hold` → `pending_payment` (места для других ещё доступны).
2. Шаг оплаты → Stripe Checkout Session (живёт до 24 ч).
3. **Оплатил** → webhook `checkout.session.completed` → `confirmed` → письма/SMS → при sold out закрываем остальных.
4. **Круиз заполнился пока человек платил** → polling на фронте + `sold_out` → форма показывает «Fully booked».
5. **Через 24 ч не оплатил** → Stripe `checkout.session.expired` → бронь `Истекло удержание`.
6. **Запасной cleanup** (Celery каждые 30 мин) — если webhook не пришёл, pending всё равно помечается истёкшей.

## Webhook events от Stripe

| Событие | Что делаем |
|---------|------------|
| `checkout.session.completed` | Бронь подтверждена, письма/SMS, при необходимости sold-out sweep |
| `checkout.session.expired` | Эта бронь истекла (человек не оплатил за 24 ч в Stripe) |
